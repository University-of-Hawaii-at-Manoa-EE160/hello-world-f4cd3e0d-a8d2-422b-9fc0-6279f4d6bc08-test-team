    summer.c:

    Things to note:

    • control statements can be nested
    • nesting control statements too deeply will make the code hard to read
    • if you need to nest deeply, then consider using functions to abstract away some of the logic; we'll discuss functions and abstraction in the next lecture

    Be careful with the counter:

    • counter = 1;
    • while (counter <= MAX_NUMBER_OF_VALUES)
    • if (counter == MAX_NUMBER_OF_VALUES)
    • be sure the offset is correct to get the results you expect
    • Progression of counter, with MAX_NUMBER_OF_VALUES = 3
      loop iteration value of counter at the beginning of the loop while (counter <= 3) if (counter == 3) value of counter at the end of the loop
      1 1 while (1 <= 3)
      =while (true)      		 if (1 == 3)
      =if (false)      		 2
      2 2 while (2 <= 3)
      =while (true)      		 if (2 == 3)
      =if (false)      		 3
      3 3 while (3 <= 3)
      =while (true)      		 if (3 == 3)
      =if (true)      		 4
      4 4 while (4 <= 3)
      =while (false)      		 if statement never gets executed counter never gets incremented

    Another implementation:

    • counter = 0;
    • while (counter < MAX_NUMBER_OF_VALUES)
    • if (counter == MAX_NUMBER_OF_VALUES - 1)
    • Progression of counter, with MAX_NUMBER_OF_VALUES = 3
      loop iteration value of counter at the beginning of the loop while (counter < 3) if (counter == 2) value of counter at the end of the loop
      1 0 while (0 < 3)
      =while (true)      		 if (0 == 2)
      =if (false)      		 1
      2 1 while (1 < 3)
      =while (true)      		 if (1 == 2)
      =if (false)      		 2
      3 2 while (2 < 3)
      =while (true)      		 if (2 == 2)
      =if (true)      		 3
      4 3 while (3 < 3)
      =while (false)      		 if statement never gets executed counter never gets incremented

  • Assignment

    write a program that:

    • asks the user for a number
    • if the number is greater than 100, then the program will output the number less 5 three times
    • if the number is less than or equal to 100, then the program will output the number to the third power

    example outputs:

    • if the user enters 101, then the outputs will be 96, 91, and 86
    • if the user enters 90, then the output will be 729000
